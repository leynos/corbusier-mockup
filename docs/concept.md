# Corbusier Concept

Corbusier is an AI agent orchestration platform built on the premise that the room matters more than the occupants. Rather than pursuing ever-larger context windows or ever-more-capable single agents, it provides the coordination layer — conversation history, task lifecycle, tool surface, governance hooks, and VCS primitives — that allows multiple specialised agent backends to work within a shared, auditable workflow. The core is a Rust modular monolith organised as hexagonal architecture: domain logic behind ports, infrastructure behind adapters, everything connected through dependency inversion. Claude Code SDK and OpenAI's Codex CLI plug in as interchangeable backend adapters; the Model Context Protocol provides a uniform tool plane. The architecture deliberately avoids distributed-systems complexity in favour of clean module boundaries that can be extracted into services later if scaling demands it.

The domain model centres on three aggregates. Conversations store append-only message sequences in a canonical format — user, assistant, tool, and system roles with typed content parts — preserving full audit metadata including tool-call records and agent-response provenance. Tasks track the lifecycle from external issue reference through branch association to pull request, enforcing a state machine (`draft` → `in_progress` → `in_review` → `done` | `abandoned`, with `paused` as an escape hatch) via typed transition validation that rejects illegal moves at the domain boundary. MCP server registrations manage the tool plane: servers are registered, started, health-checked, and queried for their tool catalogs, with lifecycle transitions (`registered` → `running` → `stopped`) enforced in the aggregate. Row-level multi-tenancy partitions all state by `tenant_id`, with PostgreSQL RLS as a hard guardrail behind the Rust-level scoping — defence in depth against query bugs silently becoming cross-tenant leaks.

The front end, mapped across forty-odd routes, gives operational shape to these abstractions. A persistent sidebar organises navigation into three tiers: a personal mainframe (dashboard, task queue, AI-generated suggestions), a project-scoped workspace (kanban boards, conversation threads, dependency hierarchies, slash command registries), and a system administration plane (agent backend registry, MCP tool management, hook configuration, monitoring dashboards, tenant controls). Task detail pages are the densest surface — showing the full dependency hierarchy from goal through milestone step to current task, progress tracking, subtask checklists, upstream and downstream blockers, branch and PR associations, and a chronological audit timeline. Conversation views render canonical message history as a chat interface with inline tool-execution cards, agent-status badges, slash-command auto-completion, and handoff annotations marking where the backing agent changed mid-conversation. All live views subscribe to server-sent events filtered by scope, so state transitions propagate to every open browser tab without polling.

The governance layer, still largely ahead on the roadmap, will wire hook definitions to trigger points across the workflow — pre-commit gates, policy checks on tool calls, quality gates before PR creation — with structured outcomes feeding back into conversation context so agents can self-correct. Slash commands already produce deterministic tool-call plans with SHA-256-derived call identifiers, making execution reproducible and auditable. The design spec envisions Weaver as the sole authorised file editor, preventing agent backends from making stealth modifications outside the change-tracking system. Podbot containers provide workspace encapsulation, isolating tool execution behind network policies and resource limits. The combination — hooks, Weaver-only edits, containerised execution, tenant-scoped RLS — forms a layered security model where no single bypass compromises the whole.

What distinguishes Corbusier from the growing field of agent frameworks is the deliberate inversion of priorities: workflow ownership over agent capability, governance over autonomy, auditability over speed. The front end makes this philosophy tangible — the AI Suggestions page doesn't just list tasks, it shows confidence scores and dependency rationale so a human can evaluate the recommendation rather than rubber-stamp it. The task dependency view doesn't hide the complexity of multi-step engineering work behind a flat backlog; it renders the full hierarchy so blocking relationships are visible at a glance. The monitoring and hook-execution views treat observability as a first-class product surface, not an ops afterthought. Corbusier bets that as agent capabilities commoditise, the organisations that win will be the ones with the best orchestration, not the smartest individual agent — and the platform is designed, from domain model to pixel, to make that bet pay off.
