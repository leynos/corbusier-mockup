Punch-Card Chamfer

The clipped top-right corner from mainframe punch cards. On an 80-column card, the chamfer tells the operator which way to feed it — orientation at a glance. An inline SVG polygon draws both the border and the background fill. A ResizeObserver recomputes polygon points in pixel coordinates so the chamfer angle stays at exactly 45° regardless of the card's aspect ratio. vector-effect: non-scaling-stroke keeps the border at 1 CSS pixel. Works cross-browser without feature-detection fallbacks.

Orientation semantics

Top-right bevel. Card feeds correctly — work is moving forward. SVG polygon frame stretches to fill.

Reversed orientation
Top-left bevel. Card is backwards — something is wrong. Visible discomfort by design.
⊘ Blocked TASK-331

Where it applies

The chamfer is for cards and machine-readable surfaces: task cards (Kanban and detail), code blocks, slash command expansions, tool call output. It does not apply to buttons (breaks target shape), badges (too small), form inputs (fights native affordance), or information cards that aren't moveable work units.

Foundation

Design Principles

The room is the product
The interface is an operations room. Wall-mounted boards, index cards, message logs. Every panel earns its place by answering a question.
Warmth from materials
Verdigris, terracotta, olive, cream — the surfaces themselves are the aesthetic. Colour carries meaning; materials carry trust.
Structure as ornament
Visible grid, exposed system language, architectural wayfinding. The load-bearing walls are the decoration. Nothing cosmetic that doesn't work.
